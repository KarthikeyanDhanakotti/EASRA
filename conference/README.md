# EASRA Conference & Community Materials

This folder holds materials that let architects, engineers, and educators bring EASRA to conferences, meetups, workshops, and training programmes. All materials are dual-licensed under CC-BY-4.0 (docs) and Apache-2.0 (code) — the same as the rest of the repository — and may be reused with attribution.

## Suggested talk outlines

### Introductory keynote (30 min)

1. Why Enterprise AI needs a reference architecture (5 min).
2. The 16 EASRA layers and the 5 architecture views (10 min).
3. Capability model + component catalogue (5 min).
4. A single-agent RAG walk-through against EASRA (5 min).
5. How to adopt (5 min).

### Deep-dive (60 min)

1. Layered model + design principles (10 min).
2. Trust boundaries and threat model (10 min).
3. Guardrails vs. verification vs. evaluation (10 min).
4. Runtime execution flow — decision points (10 min).
5. LLMOps: evaluation-gated release (10 min).
6. Q & A (10 min).

### Enterprise architect briefing (60 min)

1. EASRA as an open architecture standard (10 min).
2. Standards mapping: NIST AI RMF, ISO 42001, EU AI Act, OWASP LLM (15 min).
3. Adoption paths and the 5-level capability maturity model (10 min).
4. Governance, ADR process, deprecation policy (10 min).
5. Q & A (15 min).

### Half-day workshop (4 h)

- Module 1 — Map an existing system to EASRA layers (60 min).
- Module 2 — Identify capability gaps and set target maturity (60 min).
- Module 3 — Trust boundary and threat exercise (60 min).
- Module 4 — Author a capability maturity statement (60 min).

## Contributing materials

1. Choose a slot from the table above (or open an issue proposing a new one).
2. Land the material under the appropriate subfolder.
3. Include speaker notes and licence line in each file.
4. If you deliver the talk publicly, add a row to `deliveries.md` (planned) with venue, date, and — if recorded — the URL.
