# Checklist — Repository Quality

**Purpose.** The bar every repository in the EASRA ecosystem (EASRA itself, future EAEC, EAVF, CognitiveAgentLab, etc.) MUST meet before it is announced or pinned.

**When to use.** Before creating a new repository. Before publishing a repository. Before pinning to a profile.

## Items

Mark each item **Pass**, **Fail**, or **N/A** with a one-line justification.

- [ ] **1. README with elevator pitch** — first paragraph states what, for whom, and why in ≤ 3 sentences.
- [ ] **2. Architecture section or diagram** — even a single Mermaid diagram; no diagram-less repo.
- [ ] **3. Quick Start** — a copy-pasteable path from clone to first result in ≤ 5 commands.
- [ ] **4. Screenshots or example output** — at least one visual of what the repo does.
- [ ] **5. Diagrams folder or inline diagrams** — every named component appears in a diagram.
- [ ] **6. Roadmap** — phased plan with checkable items, no dates.
- [ ] **7. Contributing guide** — how to propose changes, run tests, and open ADRs.
- [ ] **8. Governance (if reference/standard)** — versioning, deprecation, ADR process.
- [ ] **9. License** — explicit; dual CC-BY-4.0 (docs) + Apache-2.0 (code) for standard-family repos.
- [ ] **10. Examples folder** — at least one runnable worked example.
- [ ] **11. References** — citations to specs, standards, and prior art.
- [ ] **12. Research or ADRs** — a place for forward-looking / non-shipping work.
- [ ] **13. Changelog** — Keep-a-Changelog format.
- [ ] **14. CI workflow** — at minimum markdown-lint on every push and PR.
- [ ] **15. Issue templates + PR template** — under `.github/`.
- [ ] **16. Topics set** — ≥ 5 GitHub topics describing the repository.
- [ ] **17. Description + website field** — non-empty on GitHub.
- [ ] **18. Discussions enabled** (if community-facing).
- [ ] **19. Consistent visual identity** — matches the ecosystem style guide.
- [ ] **20. Cross-links to sibling repos** — profile-level coherence.

## Scoring

- **≥ 18 Pass** → Publishable and pinnable.
- **14–17 Pass** → Public but not pinned; open issues for the gaps.
- **< 14 Pass** → Keep private until closed.
