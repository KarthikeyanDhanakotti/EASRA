## 1. Purpose

This specification defines the categories, targets, and layer attachments for the non-functional requirements (NFRs) that a spec-conformant EASRA system must address.

EASRA does **not** prescribe absolute NFR values (they depend on domain, cost budget, and user expectations). It prescribes:

- The **categories** every system must define targets for.
- The **layer** each NFR attaches to.
- The **telemetry** required to measure it.
- The **conformance rule** (what "having" the NFR means).

## 3. N1 — Latency

**What must be defined.** Per-mode latency SLOs (Simple Chat, RAG, Single-Agent, Multi-Agent) at P50, P95, P99.

**Layer breakdown (required).** Latency budget allocated across L1, L3/L4/L5, L6, L7, L8, L9.

**Telemetry.** Per-layer span duration; end-to-end request latency; cache-hit vs cache-miss latency; streaming time-to-first-token.

**Conformance rule.** Every request emits a latency breakdown per layer; SLO breaches trigger an alert.

## 4. N2 — Throughput

**What must be defined.** Sustained requests-per-second per tenant and per platform; burst capacity; queuing behaviour.

**Layer breakdown.** Gateway capacity (L1), orchestrator parallelism (L2), model provider RPS/TPM ceilings (L6), cache capacity (L11).

**Telemetry.** Per-layer RPS; per-tenant RPS; queue depth; back-pressure signals.

**Conformance rule.** Load tests demonstrate stated throughput; back-pressure is graceful (429 with `retry-after`), not silent drops.

## 5. N3 — Availability

**What must be defined.** SLA per component and per platform (e.g., 99.9% end-to-end); dependency SLO stack; degraded-mode definitions.

**Layer breakdown.** Every external dependency has a stated SLO and a degraded-mode.

**Telemetry.** Uptime per component; error budget consumption; degraded-mode entry/exit events.

**Conformance rule.** Every layer has a documented degraded mode (P8). Loss of one dependency does not take down the platform.

## 6. N4 — Reliability

**What must be defined.** Retry policies, circuit-breaker thresholds, timeout budgets, idempotency guarantees, at-most-once vs at-least-once semantics per tool.

**Layer breakdown.** L6 (model retries), L7 (tool retries and idempotency), L11 (cache write consistency), L4 (memory write consistency).

**Telemetry.** Retry counts, circuit-breaker state, timeout events, duplicate-detection metrics.

**Conformance rule.** Every external call has a timeout and a circuit breaker (P8).

## 7. N5 — Scalability

**What must be defined.** Horizontal scaling model (P6); autoscaling triggers; scale-out and scale-in latency; multi-tenant isolation.

**Layer breakdown.** Compute layers scale horizontally; data-plane stores scale per store type.

**Telemetry.** Instance count, CPU/memory saturation, scaling events.

**Conformance rule.** Any compute instance is disposable; killing one instance does not fail a request unrecoverably.

## 8. N6 — Cost

**What must be defined.** Per-request cost ceiling; per-tenant monthly ceiling; per-agent budget; cost-attribution taxonomy.

**Layer breakdown.** Token cost (L6), storage cost (L4/L5), cache savings (L11), value attribution (L15).

**Telemetry.** Per-request cost; per-tenant cost; cost saved by cache; cost per business outcome.

**Conformance rule.** Every model call emits token counts and cost (P9). Cost is queryable per tenant, user, agent, request.

## 9. N7 — Safety

**What must be defined.** Guardrail policy set; verification check set; escalation matrix; harm taxonomy.

**Layer breakdown.** L8 (guardrails), L9 (verification), L14 (policy).

**Telemetry.** Guardrail decisions and reasons; verdict per check; escalation events; safety-incident rate.

**Conformance rule.** Every response carries a verdict (P3). Every escalation is audit-logged.

## 10. N8 — Security

**What must be defined.** Trust boundary controls (Spec 009); threat model; incident response playbook; security-SLOs (patch time, MTTR).

**Layer breakdown.** L13 cross-cutting; enforcement at TB-A, TB-B, TB-C, TB-D, TP-1, TP-2.

**Telemetry.** Authn/authz events; policy decisions; secret access; audit log integrity.

**Conformance rule.** Every OWASP LLM Top 10 threat maps to a named control at a named boundary (Spec 009 §9).

## 11. N9 — Privacy

**What must be defined.** Data classification taxonomy (Spec 007 §15); retention per class; data-subject rights process; DPIA if applicable.

**Layer breakdown.** L4 memory, L5 knowledge, L11 caches, L10 observability, L13 controls.

**Telemetry.** PII detection events; redaction rate; retention-policy adherence.

**Conformance rule.** PII is classified on write; observability redacts PII before persistence; no cache key contains PII in cleartext.

## 12. N10 — Observability

**What must be defined.** Signal set per layer (traces, metrics, logs, AI-specific); sampling policy; retention per signal; alert catalogue.

**Layer breakdown.** L10 owns the plane; every layer emits.

**Telemetry.** Meta-signals: collector health, buffer depth, drop rate.

**Conformance rule.** Every canonical sequence in Spec 008 has an end-to-end trace and complete AI-specific telemetry (P4).

## 13. N11 — Maintainability

**What must be defined.** Change-lead-time SLO; test coverage per layer; deployment frequency; MTTR.

**Layer breakdown.** L12 owns the discipline; every layer contributes.

**Telemetry.** DORA metrics (lead time, deployment frequency, change-failure rate, MTTR).

**Conformance rule.** No prompt or model reaches production untested; every deployed artefact is traceable to a commit.

## 14. N12 — Portability

**What must be defined.** Vendor mapping matrix (per layer, ≥ 2 implementations); interface conformance (no proprietary types in Spec 006 signatures).

**Layer breakdown.** All layers (P1).

**Telemetry.** N/A (design-time property).

**Conformance rule.** Any layer's implementation can be swapped with a spec-conformant alternative in a bounded time window (declared per platform).

## 15. N13 — Compliance

**What must be defined.** Applicable standards (GDPR, HIPAA, SOC 2, ISO/IEC 42001, EU AI Act, industry-specific); control mapping; audit cadence.

**Layer breakdown.** L14 cross-cutting.

**Telemetry.** Policy decisions; audit-log integrity; compliance dashboards.

**Conformance rule.** Every applicable standard is mapped to controls in Spec 009 §10 or the handbook.

## 16. N14 — Accessibility

**What must be defined.** WCAG level (recommended 2.2 AA); alternative modalities (screen readers, voice); language and localisation.

**Layer breakdown.** L0 owns.

**Telemetry.** Client-side accessibility metrics.

**Conformance rule.** L0 channel adapters meet the stated accessibility level.

## 17. Cross-NFR trade-offs

| Trade-off | Resolution |
|-----------|------------|
| Latency vs Verification depth | Verification cost bounded per policy (P9). Lower-value requests get lighter verification. |
| Cost vs Availability | Fallback models may be cheaper; fallback policy is explicit. |
| Safety vs Latency | Guardrails run in parallel where possible; safety failures short-circuit. |
| Observability vs Privacy | PII is redacted before persistence; sampling reduces volume without dropping outliers. |
| Portability vs Cost | Portability may forgo vendor-specific optimisations; the trade is explicit per layer. |

## 18. NFR review cadence

- NFR targets are reviewed **quarterly** against production data.
- Breaches drive a *review* (why did this happen?) and either a *target revision* (with justification) or a *design change* (via ADR).
